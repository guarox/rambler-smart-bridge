Step-by-Step Diagnostic & Configuration Guide

Follow these steps while aboard the boat with the Pi turned on and connected to the NMEA 2000 network (your B&G instruments must be powered up).

  1. Connect to the Pi's Network:
    • Connect your laptop or iPad to the boat's Wi-Fi: Rambler_Net (Password: RAMBLER2026).
    • Open a terminal (macOS/Linux) or a Terminal App (iOS) and SSH into the Pi: 
      ssh guarox@rambler.local
# Password: 2wsx#edC
  2. Check the OS Network Interface:

    Verify if the CAN interface is enabled and up in the Linux kernel:

    ip link show can0

    Expected Output (Success):

    can0: <NOARP,UP,LOWER_UP,ECHO> mtu 16 qdisc pfifo_fast state UP mode DEFAULT group default qlen 10
    link/can

    ⚠️ If it says state DOWN or doesn't show UP, run the updated pi_setup.sh script or bring it up manually using: sudo ip link set can0 up type can bitrate 250000.

  3. Check Raw Data Flow (candump):

    Test if the physical CAN transceiver is receiving bytes from the NMEA 2000 backbone:

    candump can0

    Expected Output:

    can0  0df01000   [8]  01 02 03 04 05 06 07 08
can0  09f80100   [8]  ff 7f ff ff 00 1a 2b 3c
... hex packets scrolling continuously ...
    💡
    If candump shows nothing: 1. Verify that your B&G instrument panel is powered ON. The PICAN-M transceivers require 12V bus power from the NMEA 2000 backbone to operate.
    2. Verify termination: Ensure you have exactly two 120Ω resistors at the physical ends of your NMEA 2000 bus.
  4. Configure Signal K Connection (First-Time Only):

    Once raw CAN frames are verified, you must tell Signal K to ingest this interface.

    • Open your browser and navigate to: http://rambler.local:3000
    • Log in (create an administrator account if it is the first run).
    • Go to Server > Data Connections in the left sidebar.
    • Click Add Connection and input:
      ActiveYes
      IDcan0
      Input TypeNMEA 2000
      NMEA 2000 SourceCanbus (canboatjs)
      Interfacecan0
    • Click Save and restart the Signal K server when prompted.

Troubleshooting Matrix

Symptom Root Cause Command / Fix
ip link does not list can0 Device tree overlay is not loaded in configuration. Check that /boot/firmware/config.txt contains:
dtoverlay=mcp2515-can0,oscillator=16000000,interrupt=25
can0 exists but is state DOWN systemd-networkd has not run, or network config is missing. Verify configuration exists at:
/etc/systemd/network/80-can.network
Then run:
sudo systemctl restart systemd-networkd
candump can0 prints error: Network is down CAN driver is not brought up. Bring the link up manually:
sudo ip link set can0 up type can bitrate 250000
candump is empty (instruments ON) No 12V power on NMEA 2000 backbone, or wiring mismatch. Measure voltage between Red (+) and Black (-) wires on your NMEA 2000 backbone. It must be between 11.8V and 13.8V.
Signal K connects, but dashboard shows — values Signal K is not parsing N2K or connection was not created. Check Signal K dashboard at :3000. Verify that "Data Connections" lists an active `can0` interface receiving updates.

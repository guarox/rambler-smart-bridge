Weather Ingestion & Routing Reference

⛈ NOAA HRRR GRIB Grid Ingestion

The Pi downloads high-resolution (3-km) wind forecast grids automatically when internet is available:

  1. Starlink WAN connection is detected via the PoE gateway interface.
  2. A script on the Pi runs download_hrrr.py to download a filtered GRIB2 from NOAA NOMADS.
  3. It isolates U/V wind vectors at 10m height for the Great Lakes area (under 150KB size).
  4. It decodes the grid into /home/guarox/gribs/hrrr_latest.json for 100% offline access at the helm.

Boat Installation Checklist

Check off steps on your tablet as you proceed. Progress is saved automatically to your device storage.

🔧 Stage 1: Hardware Pre-assembly

  • Mount PICAN-M HAT securely to the Pi 4 using standard brass spacers.
  • Assemble passive Armor cooling case, aligning heat pads on Pi 4 CPU and RAM.
  • Insert flashed 128GB SanDisk MAX Endurance MicroSD card into the Pi 4.
  • Prepare two short-run CAT6 Ethernet cables (Pi to XTAR-Link).

⛵ Stage 2: Boat Integration (First Run)

  • Connect 12V House Battery positive line to Blue Sea 4153 Switch + 5A fuse.
  • Run wire from switch outputs to PICAN-M HAT screw terminals (V+ & GND).
  • Connect micro-C NMEA 2000 drop cable from PICAN-M to NSPL-500 Hub.
  • Power ON the system. Verify red PWR indicator on enclosure lights up.
  • Connect iPad to Rambler_Net (password: RAMBLER2026).
  • Open http://rambler.local:3000 and create admin account.
  • Go to Server -> Data Connections -> Add -> NMEA 2000 / can0. Save & Restart.
  • Set vessel identity: Name: Rambler, MMSI: 338380946.

🏁 Stage 3: Race Day Checklist

  • Turn on Pi (Blue Sea 4153 switch ON). Wait 45 seconds for full initialization.
  • Verify iPad Helm Display is connected to SSID: Rambler_Net.
  • Open browser to http://192.168.5.1. Verify dashboard loads.
  • Connect Signal K: Click indicator badge → enter 192.168.5.1:3000 → Connect.
  • Power on B&G array. Check that telemetry badge goes ● LIVE.
  • Set start marks: Drop committee boat (Start P) and pin end (Start S).
  • Tap ▶ Start Race to begin recording logs to local storage.

Troubleshooting Matrix

Type in the box below to filter issues by symptoms (e.g. "wifi", "candump", "down", "cert").

Symptom / Error Root Cause Analysis Resolution Commands & Procedures
ip link does not show can0 Device tree overlay has not been loaded in the boot partition configuration. Verify /boot/firmware/config.txt contains:
dtoverlay=mcp2515-can0,oscillator=16000000,interrupt=25
can0 exists but is state DOWN The systemd network unit is missing, or the service has not run. Verify configuration exists at:
/etc/systemd/network/80-can.network
Then run:
sudo systemctl restart systemd-networkd
candump can0 outputs Network is down CAN drivers are loaded in the kernel but the link state was not brought up. Manually trigger link interface initialization:
sudo ip link set can0 up type can bitrate 250000
candump is empty (instruments are ON) No 12V bus power detected on the NMEA 2000 backbone, or wiring polarity is reversed. Verify termination: measure exactly ~60Ω across NET+ & NET- with power OFF.
Measure voltage between Red (+) and Black (-) wires. Must be 11.8V to 13.8V.
iPad cannot locate Rambler_Net AP SSID WiFi chip interface is soft-blocked by rfkill, or hostapd daemon crashed. SSH in (wired) or console to unblock interface:
sudo rfkill unblock wifi
sudo systemctl restart hostapd
Signal K connects, but helm dashboard shows — N2K data connections are not configured in the Signal K Admin panel. Log in to http://192.168.5.1:3000. Go to Server → Data Connections. Add can0 as Canbus (canboatjs).
Dashboard displays ◌ SK… and refuses to connect Signal K daemon is offline or port 3000 is blocked. Restart the service container daemon:
sudo systemctl restart signalk
Tablet gets no IP when connecting to AP The DHCP server daemon (dnsmasq) is offline or misconfigured. Check log outputs and restart DHCP allocator daemon:
sudo systemctl restart dnsmasq
HTTPS certificate error when loading dashboard Attempting to load the dashboard via HTTPS local page without SSL certificate keys. Always load local dashboard using plain HTTP:
http://192.168.5.1 (not https)
